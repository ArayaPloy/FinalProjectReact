## 1. ภาพรวมของระบบ

ระบบจัดการเว็บไซต์และสารสนเทศโรงเรียนเป็นระบบเว็บแอปพลิเคชันแบบครบวงจร ครอบคลุมการแสดงข้อมูลสาธารณะของโรงเรียน การบริหารจัดการนักเรียน การเช็คชื่อเข้าแถว การบันทึกคะแนนความประพฤติ การเยี่ยมบ้านนักเรียน และการจัดการบทความข่าวสาร

### กลุ่มผู้ใช้งาน

| กลุ่มผู้ใช้ | บทบาท (Role) | รายละเอียดสิทธิ์ |
|---|---|---|
| บุคคลทั่วไป / ผู้ปกครอง / นักเรียน | — (ไม่ต้องล็อกอิน) | ดูข้อมูลสาธารณะ ตรวจสอบการเช็คชื่อและคะแนนความประพฤติ |
| ครู / บุคลากร | teacher | เช็คชื่อ บันทึกคะแนนพฤติกรรม บันทึกการเยี่ยมบ้าน เขียนบทความ |
| ผู้ดูแลระบบ | admin | จัดการข้อมูลทั้งหมด อนุมัติบทความ จัดการบัญชีผู้ใช้ ดูรายงาน |
| ผู้ดูแลระบบสูงสุด | super_admin | สิทธิ์ทุกอย่างของ admin รวมถึงการจัดการข้อมูลระดับระบบ |

## 2. สถาปัตยกรรมและ Flow การทำงาน

### 2.1 โครงสร้างระบบ

ระบบแบ่งเป็น 3 ส่วนหลัก:

- Frontend — React.js + Redux Toolkit + Tailwind CSS (รันที่ port 5173)
- Backend — Node.js + Express.js + Prisma ORM (รันที่ port 5000)
- Database — MySQL (เก็บข้อมูลทั้งหมด)

### 2.2 Flow การยืนยันตัวตน (Authentication Flow)

```
ผู้ใช้กรอก Username / Password
        ↓
Backend ตรวจสอบข้อมูลกับ Database
        ↓
หากถูกต้อง → ออก JWT Token (เก็บใน HttpOnly Cookie)
        ↓
Frontend อ่าน Token จาก Cookie → บันทึกข้อมูลผู้ใช้ใน Redux Store
        ↓
ทุก API Request จะแนบ Cookie ไปโดยอัตโนมัติ (credentials: include)
        ↓
Backend Middleware ตรวจสอบ Token → อนุญาตหรือปฏิเสธ Request
```

### 2.3 Flow การควบคุมสิทธิ์ (Authorization Flow)

```
ผู้ใช้เข้าหน้าเว็บ
        ↓
ProtectedRoute ตรวจสอบ Token และ Role
        ↓
Role ตรงตามที่กำหนด → แสดงหน้า
Role ไม่ตรง / ไม่มี Token → Redirect ไปหน้า Login
```

หน้าที่กำหนดสิทธิ์เฉพาะ:

| หน้า | บทบาทที่อนุญาต |
|---|---|
| เช็คชื่อนักเรียนเข้าแถว | teacher, admin, super_admin |
| บันทึกคะแนนความประพฤติ | teacher, admin, super_admin |
| บันทึกการเยี่ยมบ้าน | teacher, admin, super_admin |
| Admin Dashboard และทุกหน้าใต้ /dashboard | admin, super_admin |

### 2.4 Flow การบันทึกข้อมูลทั่วไป

```
ผู้ใช้กรอกข้อมูลในฟอร์ม
        ↓
Frontend Validate ข้อมูลเบื้องต้น
        ↓
ส่ง HTTP Request (POST/PUT) ไป Backend API
        ↓
Backend ตรวจสอบความถูกต้องและ Middleware
        ↓
บันทึกข้อมูลลง Database ผ่าน Prisma
        ↓
ส่ง Response กลับ → แสดงผลแจ้งเตือนบนหน้าจอ
```

---

### 4.1 หน้าหลัก (Home)

URL: `/`

หน้าหลักแสดงข้อมูลภาพรวมของโรงเรียน ได้แก่ ข่าวล่าสุด กิจกรรม และลิงก์ไปยังส่วนต่าง ๆ ของเว็บไซต์

ขั้นตอน:
1. เปิดเว็บเบราว์เซอร์และพิมพ์ที่อยู่เว็บไซต์โรงเรียน
2. ระบบแสดงหน้าหลักโดยอัตโนมัติ
3. ใช้เมนูนำทางด้านบนเพื่อไปยังหน้าต่าง ๆ

### 4.2 ข้อมูลโรงเรียน

#### 4.2.1 ประวัติโรงเรียน

URL: `/about-us`

แสดงประวัติความเป็นมา ข้อมูลทั่วไป และสัญลักษณ์ของโรงเรียน

#### 4.2.2 วิสัยทัศน์และพันธกิจ

URL: `/vision-mission`

แสดงวิสัยทัศน์ พันธกิจ และเป้าหมายการพัฒนาของโรงเรียน

#### 4.2.3 โครงสร้างสายงาน

URL: `/school-board`

แสดงแผนผังโครงสร้างการบริหารและรายชื่อคณะกรรมการสถานศึกษา

#### 4.2.4 คณะครูและบุคลากร

URL: `/faculty-staff`

แสดงรายชื่อครูและบุคลากรทั้งหมด จัดกลุ่มตามกลุ่มสาระการเรียนรู้ ได้แก่ คณะผู้บริหาร, กลุ่มสาระภาษาไทย, คณิตศาสตร์, วิทยาศาสตร์และเทคโนโลยี, สังคมศึกษา, สุขศึกษา, ศิลปะ, ภาษาต่างประเทศ, ธุรการ และนักการภารโรง

ขั้นตอน:
1. คลิกเมนู "เกี่ยวกับโรงเรียน" แล้วเลือก "บุคลากรและเจ้าหน้าที่"
2. ระบบแสดงรายชื่อครูพร้อมรูปภาพ ตำแหน่ง และข้อมูลการติดต่อ
3. คลิกที่รายชื่อครูเพื่อดูรายละเอียดเพิ่มเติม

1. คลิกเมนู "ข่าวและกิจกรรม"
2. ระบบแสดงรายการบทความทั้งหมดพร้อมรูปปก หัวข้อ และวันที่
3. สามารถกรองบทความตามหมวดหมู่ได้จากแถบด้านบน

3. เลื่อนลงเพื่อดูและแสดงความคิดเห็น (ต้องเข้าสู่ระบบจึงจะแสดงความคิดเห็นได้)

### 4.6 การตรวจสอบประวัติการเช็คชื่อเข้าแถว

URL: `/student-attendance-history`

ผู้ปกครองและนักเรียนสามารถตรวจสอบประวัติการเข้าแถวได้โดยไม่ต้องเข้าสู่ระบบ

Flow การทำงาน:
```
ผู้ใช้กรอกรหัสนักเรียน + ช่วงวันที่
        ↓
ระบบค้นหาข้อมูลการเช็คชื่อจาก Database
        ↓
แสดงผลรายวัน: สถานะแต่ละวัน (มา / ขาด / ลาป่วย / ลากิจ / สาย)
        ↓
แสดงสรุปสถิติ: จำนวนวันรวมแต่ละสถานะ
```

ขั้นตอน:
1. คลิกเมนู "กิจกรรมนักเรียน" แล้วเลือก "เช็คประวัติการเข้าแถว"
2. กรอกรหัสนักเรียนในช่องค้นหา
3. เลือกวันเริ่มต้นและวันสิ้นสุดที่ต้องการตรวจสอบ
4. คลิกปุ่ม "ค้นหา"
5. ระบบแสดงตารางประวัติการเช็คชื่อรายวัน พร้อมสรุปสถิติรวม

สถานะการเช็คชื่อที่ระบบรองรับ:

| สถานะ | ความหมาย |
|---|---|
| มา | นักเรียนมาเข้าแถวตามปกติ |
| สาย | นักเรียนมาช้ากว่าเวลาที่กำหนด |
| ลาป่วย | นักเรียนลาหยุดเนื่องจากเจ็บป่วย |
| ลากิจ | นักเรียนลาหยุดด้วยเหตุจำเป็น |
| ขาด | นักเรียนไม่มาโดยไม่แจ้ง |

หมายเหตุ: หากไม่พบข้อมูล ให้ตรวจสอบว่ากรอกรหัสนักเรียนถูกต้อง และช่วงวันที่ครอบคลุมระยะเวลาที่ต้องการ

ขั้นตอน:

### 5.1 การเข้าสู่ระบบ

URL: `/login`

Flow การทำงาน:
```
ผู้ใช้กรอก Username + Password
        ↓
Backend ตรวจสอบข้อมูลและสถานะบัญชี
        ↓ (ถูกต้อง)
ออก JWT Token → เก็บใน HttpOnly Cookie
บันทึก lastLogin → Redirect ไปหน้าหลักครู
        ↓ (ผิดพลาด)
แสดงข้อความแจ้งเตือน
```

ขั้นตอน:
1. เปิดเว็บไซต์โรงเรียน
2. คลิกปุ่ม "เข้าสู่ระบบ" มุมบนขวาของหน้า
3. กรอกชื่อผู้ใช้ (username) และรหัสผ่าน
4. คลิกปุ่ม "เข้าสู่ระบบ"
5. ระบบ Redirect ไปหน้าหลักโดยอัตโนมัติ

หมายเหตุ:
- หากเข้าสู่ระบบครั้งแรกและผู้ดูแลระบบตั้งรหัสผ่านชั่วคราวไว้ ระบบจะแจ้งให้เปลี่ยนรหัสผ่านก่อนใช้งาน
- หากกรอกรหัสผ่านผิดเกิน 5 ครั้งภายใน 15 นาที ระบบจะจำกัดการเข้าถึงชั่วคราว กรุณารอสักครู่แล้วลองใหม่

---

### 5.2 การขอรีเซ็ตรหัสผ่าน (กรณีลืมรหัสผ่าน)

URL: `/forgot-password`

Flow การทำงาน:
```
ครูส่งคำขอรีเซ็ต (กรอก Username)
        ↓
ระบบบันทึกคำขอ สถานะ "pending"
        ↓
Admin เห็นแจ้งเตือนใน Dashboard
        ↓
Admin อนุมัติ → ระบบออกรหัสผ่านชั่วคราว สถานะ "approved"
Admin ปฏิเสธ → สถานะ "rejected" ต้องส่งคำขอใหม่
        ↓ (กรณีอนุมัติ)
ครูเข้าสู่ระบบด้วยรหัสผ่านชั่วคราว
        ↓
ระบบบังคับให้เปลี่ยนรหัสผ่านใหม่ทันที
```

ขั้นตอน:
1. ที่หน้าเข้าสู่ระบบ คลิก "ลืมรหัสผ่าน?"
2. กรอกชื่อผู้ใช้ (username)
3. คลิกปุ่ม "ส่งคำขอ"
4. รอให้ผู้ดูแลระบบอนุมัติและแจ้งรหัสผ่านชั่วคราว
5. เข้าสู่ระบบด้วยรหัสผ่านชั่วคราว แล้วเปลี่ยนรหัสผ่านใหม่ทันที

### 5.3 การเปลี่ยนรหัสผ่าน

URL: `/change-password`

2. เลือก "เปลี่ยนรหัสผ่าน"
3. กรอกรหัสผ่านปัจจุบัน, รหัสผ่านใหม่ และยืนยันรหัสผ่านใหม่
4. คลิกปุ่ม "บันทึก"
5. ระบบแจ้งยืนยันการเปลี่ยนรหัสผ่านสำเร็จ

หมายเหตุ: ระบบบันทึก audit log ทุกครั้งที่เปลี่ยนรหัสผ่านสำเร็จ โดยไม่บันทึกรหัสผ่านจริงไว้ในบันทึก

### 5.5 การเช็คชื่อนักเรียนเข้าแถว

URL: `/flagpoleattendance`

สิทธิ์ที่ต้องการ: teacher, admin, super_admin

Flow การทำงาน:
```
ครูเลือกห้องเรียน + วันที่
        ↓
ระบบโหลดรายชื่อนักเรียนทั้งหมดในห้องจาก Database
        ↓
ครูกำหนดสถานะให้นักเรียนแต่ละคน
        ↓
กรณีวันที่นั้นมีข้อมูลแล้ว → ระบบแจ้งเตือนให้ยืนยันก่อนบันทึกทับ
        ↓
คลิกบันทึก → ระบบส่ง POST/PUT Request ไป API
        ↓
บันทึกลง Database → แสดงข้อความยืนยันสำเร็จ
```

ขั้นตอน:
1. เข้าสู่ระบบแล้วคลิกเมนู "กิจกรรมนักเรียน" → "เช็คชื่อนักเรียนเข้าแถว"
2. เลือกห้องเรียนจากรายการ dropdown
3. เลือกวันที่ที่ต้องการเช็คชื่อ
4. ระบบแสดงรายชื่อนักเรียนทั้งหมดในห้องพร้อมปุ่มเลือกสถานะ
5. กำหนดสถานะให้นักเรียนแต่ละคน ได้แก่ มา, สาย, ลาป่วย, ลากิจ, ขาด
6. หากต้องการตั้งสถานะเดียวกันให้นักเรียนทุกคน ใช้ปุ่ม "เลือกทั้งหมดว่า..."
7. ตรวจสอบความถูกต้องแล้วคลิกปุ่ม "บันทึก"
8. ระบบแสดงข้อความแจ้งยืนยันการบันทึกสำเร็จ

หมายเหตุ:
- หากวันที่ที่เลือกมีข้อมูลการเช็คชื่ออยู่แล้ว ระบบจะแจ้งเตือนและขอการยืนยันก่อนบันทึกทับข้อมูลเดิม
- ห้องเรียนที่แสดงในรายการจะเฉพาะห้องที่มีสถานะ Active ในระบบเท่านั้น

### 5.6 การบันทึกคะแนนความประพฤติ

URL: `/behaviorscore`

สิทธิ์ที่ต้องการ: teacher, admin, super_admin

Flow การทำงาน:
```
ครูค้นหานักเรียน (ชื่อ หรือ รหัสนักเรียน)
        ↓
เลือกนักเรียนที่ต้องการบันทึก
        ↓
กรอกคะแนนที่ปรับ (ค่าเป็นลบสำหรับหักคะแนน บวกสำหรับเพิ่มคะแนน)
กรอกหมายเหตุ / รายละเอียดพฤติกรรม
        ↓
คลิกบันทึก → ระบบบันทึกลง Database
        ↓
ระบบคำนวณคะแนนรวมใหม่: 100 + ผลรวมการปรับทั้งหมด
แสดงคะแนนที่อัปเดต
```

ขั้นตอน:
1. คลิกเมนู "กิจกรรมนักเรียน" → "บันทึกคะแนนความประพฤติ"
2. เลือกห้องเรียนจาก dropdown หรือค้นหาด้วยชื่อ / รหัสนักเรียน
3. ระบบแสดงรายชื่อนักเรียนพร้อมคะแนนปัจจุบัน
4. คลิก "บันทึกคะแนน" ที่แถวนักเรียนที่ต้องการ
5. กรอกคะแนนที่ปรับ (เช่น -5 สำหรับหักคะแนน หรือ +10 สำหรับเพิ่มคะแนน)
6. กรอกหมายเหตุอธิบายพฤติกรรม
7. คลิกปุ่ม "บันทึก"
8. ระบบแสดงคะแนนรวมที่อัปเดตแล้วทันที

หมายเหตุ: ครูสามารถดูประวัติการบันทึกคะแนนของวันนั้นได้ในแท็บ "บันทึกวันนี้"

### 5.7 การบันทึกการเยี่ยมบ้านนักเรียน

URL: `/home-visits`

สิทธิ์ที่ต้องการ: teacher, admin, super_admin

Flow การทำงาน:
```
ครูค้นหานักเรียนด้วยรหัสนักเรียน
        ↓
กรอกข้อมูลการเยี่ยมบ้าน
        ↓
อัปโหลดรูปภาพหลักฐาน (สูงสุด 5 รูป)
        ↓
ระบบบันทึกข้อมูลและเก็บไฟล์รูปในเซิร์ฟเวอร์
        ↓
แสดงยืนยันการบันทึกสำเร็จ
```

ขั้นตอน:
1. คลิกเมนู "กิจกรรมนักเรียน" → "ฟอร์มเยี่ยมบ้านนักเรียน"
2. ค้นหานักเรียนด้วยรหัสนักเรียนในช่องค้นหา
3. เลือกนักเรียนจากผลการค้นหา
4. กรอกข้อมูลการเยี่ยมบ้านให้ครบถ้วน ได้แก่:
   - วันที่เยี่ยมบ้าน
   - ที่อยู่ที่เยี่ยม
   - สรุปผลการเยี่ยมบ้านและสภาพความเป็นอยู่
5. อัปโหลดรูปภาพหลักฐาน (รองรับ .jpg, .png เท่านั้น อัปโหลดได้สูงสุด 5 รูปต่อครั้ง)
6. ตรวจสอบข้อมูลแล้วคลิกปุ่ม "บันทึก"
7. ระบบแสดงข้อความยืนยันการบันทึกสำเร็จ

ข้อจำกัด:
- รองรับเฉพาะไฟล์ .jpg และ .png เท่านั้น
- อัปโหลดได้สูงสุด 5 รูปต่อการบันทึก 1 ครั้ง
- ขนาดไฟล์รวมทั้งหมดต้องไม่เกิน 10MB

### 5.8 การเขียนและจัดการบทความ

Flow การทำงานของบทความครู:
```
ครูเขียนบทความ → ส่งบทความ → สถานะ "pending" (รออนุมัติ)
        ↓
Admin รับแจ้งเตือนใน Dashboard
        ↓
Admin อนุมัติ → สถานะ "published" → บทความแสดงบนหน้าสาธารณะ
Admin ปฏิเสธ → สถานะ "rejected" → บทความไม่ถูกเผยแพร่
```

#### 5.8.1 การเขียนบทความใหม่

ขั้นตอน:
1. หลังจากเข้าสู่ระบบ คลิกที่ชื่อบัญชีผู้ใช้มุมบนขวา
2. เลือก "บทความของฉัน" หรือผ่านเมนู "ข่าวและกิจกรรม" → "เขียนบทความ"
3. คลิกปุ่ม "เขียนบทความใหม่"
4. กรอกหัวข้อบทความ
5. เลือกหมวดหมู่บทความ
6. กรอกเนื้อหาบทความในช่อง Editor
7. อัปโหลดรูปภาพปก (ถ้ามี)
8. คลิกปุ่ม "ส่งบทความ"
9. ระบบบันทึกบทความและเปลี่ยนสถานะเป็น "รออนุมัติ"
10. บทความจะแสดงบนหน้าสาธารณะก็ต่อเมื่อ Admin อนุมัติแล้วเท่านั้น

#### 5.8.2 การดูและแก้ไขบทความของตัวเอง

ขั้นตอน:
1. คลิกที่ชื่อบัญชีผู้ใช้ → "บทความของฉัน"
2. ระบบแสดงรายการบทความทั้งหมดพร้อมสถานะ (รออนุมัติ, เผยแพร่, ปฏิเสธ)
3. คลิก "แก้ไข" เพื่อแก้ไขบทความที่ยังไม่ได้เผยแพร่
4. คลิก "ลบ" เพื่อลบบทความ (ระบบขอการยืนยันก่อน)

### 5.9 การออกจากระบบ

ขั้นตอน:
1. คลิกที่ชื่อบัญชีผู้ใช้มุมบนขวา
2. เลือก "ออกจากระบบ"
3. ระบบล้างข้อมูล Session และ Cookie แล้ว Redirect กลับสู่หน้าหลัก

## 6. คู่มือสำหรับผู้ดูแลระบบ (Admin / Super Admin)

---

### 6.1 การเข้าสู่ระบบ Admin

ขั้นตอน:
1. เปิดเว็บไซต์โรงเรียน
2. คลิกปุ่ม "เข้าสู่ระบบ" มุมบนขวา
3. กรอกชื่อผู้ใช้และรหัสผ่านของบัญชี Admin
4. คลิกปุ่ม "เข้าสู่ระบบ"
5. ระบบ Redirect ไปหน้า Admin Dashboard โดยอัตโนมัติ

### 6.2 Admin Dashboard

URL: `/dashboard`

Dashboard แสดงภาพรวมของระบบ ได้แก่:

- จำนวนบทความทั้งหมด และกราฟแนวโน้มรายเดือน
- จำนวนผู้ใช้งานแยกตามบทบาท (super_admin, admin, teacher, user)
- แจ้งเตือนรายการที่ต้องดำเนินการ:
  - จำนวนบทความที่รออนุมัติ
  - จำนวนคำขอรีเซ็ตรหัสผ่านที่รอดำเนินการ
  - จำนวนผู้ใช้ใหม่ที่รอกำหนดบทบาท

Flow การแจ้งเตือน Dashboard:
```
Admin เข้าหน้า Dashboard
        ↓
ระบบโหลดข้อมูลสถิติทั้งหมด
        ↓
หากมีรายการรอดำเนินการ → แสดง Popup แจ้งเตือนอัตโนมัติ
        ↓
Admin กด "รับทราบ" → ปิด Popup
Admin กด "จัดการเลย" → Redirect ไปหน้าจัดการผู้ใช้งาน
```

### 6.3 การจัดการบัญชีผู้ใช้งาน

URL: `/dashboard/users`

#### 6.3.1 การดูรายชื่อผู้ใช้งาน

ขั้นตอน:
1. คลิกเมนู "จัดการผู้ใช้งาน" ใน Admin Sidebar
2. ระบบแสดงรายชื่อผู้ใช้ทั้งหมดพร้อมข้อมูล username, ชื่อ, บทบาท, สถานะ
3. ใช้ช่องค้นหาเพื่อหาผู้ใช้ตามชื่อหรือ username
4. กรองตามบทบาทได้จาก dropdown

#### 6.3.2 การกำหนดบทบาทผู้ใช้ใหม่

Flow การทำงาน:
```
ผู้ใช้ลงทะเบียนใหม่ → บทบาท default คือ "user"
        ↓
Admin เห็นรายชื่อในส่วน "ผู้ใช้ที่รอกำหนดบทบาท"
        ↓
Admin คลิก "แก้ไข" → เปลี่ยนบทบาทเป็น teacher, admin, หรือ super_admin
        ↓
บันทึก → ผู้ใช้ได้รับสิทธิ์ตามบทบาทใหม่ทันที
```

ขั้นตอน:
1. ในหน้าจัดการผู้ใช้งาน ค้นหาผู้ใช้ที่ต้องการกำหนดบทบาท
2. คลิกปุ่ม "แก้ไข" ที่แถวนั้น
3. เลือกบทบาทที่ต้องการจาก dropdown: teacher, admin, super_admin
4. คลิกปุ่ม "บันทึก"
5. ระบบอัปเดตบทบาทและแสดงข้อความยืนยัน

#### 6.3.3 การลบบัญชีผู้ใช้

ระบบไม่มีฟีเจอร์ระงับ/เปิดใช้งานบัญชี การจัดการบัญชีผู้ใช้ที่ไม่ต้องการสามารถทำได้โดยการลบออกจากระบบ

ขั้นตอน:
1. ค้นหาบัญชีที่ต้องการลบในตารางรายชื่อผู้ใช้
2. คลิกปุ่ม "ลบ" (สีแดง)
3. ระบบแสดง SweetAlert ขอยืนยัน คลิก "ลบผู้ใช้" เพื่อยืนยัน
4. ระบบลบบัญชีออกจากฐานข้อมูลและแสดงผลสำเร็จ

หมายเหตุ: การลบบัญชีไม่สามารถย้อนกลับได้

### 6.4 การอนุมัติ / ปฏิเสธคำขอรีเซ็ตรหัสผ่าน

URL: `/dashboard/users` (แท็บคำขอรีเซ็ต)

Flow การทำงาน:
```
ครูส่งคำขอรีเซ็ต → สถานะ "pending"
        ↓
Admin เห็นแจ้งเตือนจำนวนคำขอใน Dashboard
        ↓
Admin คลิกดูรายการคำขอ
        ↓
อนุมัติ → ระบบออกรหัสผ่านชั่วคราว สถานะเปลี่ยนเป็น "approved"
ปฏิเสธ → สถานะเปลี่ยนเป็น "rejected"
        ↓ (กรณีอนุมัติ)
Admin แจ้งรหัสผ่านชั่วคราวให้ครูผ่านช่องทางอื่น (เช่น โทรศัพท์)
        ↓
ครูเข้าสู่ระบบด้วยรหัสผ่านชั่วคราว → ระบบบังคับเปลี่ยนทันที
```

ขั้นตอน:
1. คลิกเมนู "จัดการผู้ใช้งาน" → แท็บ "คำขอรีเซ็ตรหัสผ่าน"
2. ระบบแสดงรายการคำขอที่มีสถานะ "รอดำเนินการ"
3. คลิกที่คำขอเพื่อดูรายละเอียดผู้ขอ
4. คลิกปุ่ม "อนุมัติ" หรือ "ปฏิเสธ"
5. ระบบอัปเดตสถานะคำขอทันที

2. ยืนยันการดำเนินการ
3. นักเรียนที่ถูกระงับจะไม่แสดงในรายชื่อเช็คชื่อ แต่ข้อมูลยังเก็บอยู่ในระบบ

#### 6.6.4 การนำเข้าข้อมูลนักเรียนผ่านไฟล์ CSV

Flow การทำงาน:
```
เตรียมไฟล์ CSV ตามรูปแบบที่กำหนด
        ↓
คลิกปุ่ม "นำเข้า CSV" → เลือกไฟล์
        ↓
ระบบ Parse ไฟล์ CSV (รองรับทั้ง Header ภาษาไทยและอังกฤษ)
ระบบแปลง Header ภาษาไทย → ชื่อ Field ภาษาอังกฤษโดยอัตโนมัติ
        ↓
ส่งข้อมูลไป Backend → บันทึกลง Database
        ↓
แสดงสรุปจำนวนนักเรียนที่นำเข้าสำเร็จ / ล้มเหลว
```

ขั้นตอน:
1. เตรียมไฟล์ .csv (สามารถดาวน์โหลด Template ได้จากปุ่ม "ดาวน์โหลด Template")
2. คลิกปุ่ม "นำเข้า CSV"
3. คลิก "เลือกไฟล์" และเลือกไฟล์ .csv ที่เตรียมไว้
4. คลิกปุ่ม "นำเข้า"
5. ระบบแสดงสรุปผลการนำเข้า

รูปแบบไฟล์ CSV ที่รองรับ:
- ไฟล์นามสกุล .csv เท่านั้น
- Encoding: UTF-8
- แถวแรกต้องเป็น Header (รองรับทั้ง Header ภาษาไทยและภาษาอังกฤษ)
- ตัวอย่าง Header ภาษาไทย: รหัสนักเรียน, คำนำหน้า, ชื่อ, นามสกุล, เพศ, ห้องเรียน

#### 6.6.5 การส่งออกข้อมูลนักเรียน

ขั้นตอน:
1. เลือกห้องเรียนหรือเงื่อนไขที่ต้องการ
2. คลิกปุ่ม "ส่งออก CSV"
3. ระบบดาวน์โหลดไฟล์ .csv ที่มีข้อมูลนักเรียนทั้งหมดตามเงื่อนไขที่เลือก

4. กรอกเนื้อหาบทความ

6. คลิกปุ่ม "บันทึกและเผยแพร่"
7. บทความแสดงบนหน้าสาธารณะทันทีโดยไม่รอการอนุมัติ

#### 6.10.3 การแก้ไขและลบบทความ

ขั้นตอน:
1. ในหน้าจัดการบทความ คลิกปุ่ม "แก้ไข" ที่บทความที่ต้องการ
2. แก้ไขข้อมูล แล้วคลิก "บันทึก"
3. คลิกปุ่ม "ลบ" → ยืนยันการลบ → บทความถูกลบออกจากระบบทันที

#### 6.10.4 การจัดการชุมนุมวิชาการ

URL: `/dashboard/clubs-posts`

หน้านี้ใช้สำหรับบริหารจัดการข้อมูลชุมนุมวิชาการของโรงเรียนทั้งหมด ได้แก่ การเพิ่ม แก้ไข และลบชุมนุม

Flow การทำงาน:
```
Admin เข้าหน้าจัดการชุมนุม
        ↓
ระบบโหลดรายการชุมนุมทั้งหมดจาก Database
        ↓
Admin กรองชุมนุมตามหมวดหมู่ได้จากแถบด้านบน
        ↓
เพิ่ม / แก้ไข / ลบชุมนุมตามต้องการ
        ↓
ระบบบันทึกลง Database และอัปเดตหน้าแสดงผลสาธารณะทันที
```

การดูรายการชุมนุม

ขั้นตอน:
1. คลิกเมนู "จัดการบทความชุมนุม" ใน Admin Sidebar
2. ระบบแสดงรายการชุมนุมทั้งหมดพร้อมสถิติ ได้แก่ จำนวนชุมนุมทั้งหมด, จำนวนที่เปิดรับสมัคร, และจำนวนสมาชิกรวม
3. คลิกแท็บหมวดหมู่ด้านบนเพื่อกรองชุมนุมตามประเภท เช่น ศิลปะและงานฝีมือ, วิทยาศาสตร์, คอมพิวเตอร์ ฯลฯ
4. แต่ละการ์ดชุมนุมแสดงข้อมูล: ชื่อชุมนุม, หมวดหมู่, วันและเวลาประชุม, สถานที่, ครูที่ปรึกษา, จำนวนสมาชิก/สูงสุด และสถานะการรับสมัคร

การเพิ่มชุมนุมใหม่

ขั้นตอน:
1. คลิกปุ่ม "เพิ่มชุมนุมใหม่" ด้านบนขวา
2. กรอกข้อมูลในฟอร์มให้ครบถ้วน ได้แก่:
   - ชื่อชุมนุม (จำเป็น)
   - คำอธิบายชุมนุม (จำเป็น)
   - หมวดหมู่ — เลือกจากรายการ เช่น ศิลปะและงานฝีมือ, วิทยาศาสตร์, คอมพิวเตอร์ ฯลฯ
   - ไอคอน — เลือกไอคอนที่แสดงในการ์ดชุมนุม
   - ครูที่ปรึกษา — เลือกจากรายชื่อครูในระบบ
   - จำนวนสมาชิกสูงสุด (ค่าเริ่มต้น: 20 คน)
   - วันประชุม, เวลาประชุม, สถานที่ประชุม
   - วันสิ้นสุดการรับสมัคร — หากระบุ ระบบจะคำนวณสถานะ "เปิดรับสมัคร / ปิดรับสมัคร" อัตโนมัติ
   - คุณสมบัติผู้สมัคร (ถ้ามี)
   - สถานะ Active — เปิด/ปิดการแสดงชุมนุมบนหน้าสาธารณะ
3. คลิกปุ่ม "บันทึก"
4. ระบบเพิ่มชุมนุมและแสดงในรายการทันที

การแก้ไขข้อมูลชุมนุม

ขั้นตอน:
1. คลิกไอคอน "แก้ไข" (ดินสอ) บนการ์ดชุมนุมที่ต้องการ
2. แก้ไขข้อมูลที่ต้องการในฟอร์ม
3. คลิกปุ่ม "บันทึก" เพื่อยืนยันการเปลี่ยนแปลง
4. คลิกปุ่ม "ยกเลิก" เพื่อยกเลิกโดยไม่บันทึก

การลบชุมนุม

ขั้นตอน:
1. คลิกไอคอน "ลบ" (ถังขยะ) บนการ์ดชุมนุมที่ต้องการลบ
2. ระบบแสดง SweetAlert ขอยืนยัน คลิก "ลบ" เพื่อยืนยัน
3. ระบบลบชุมนุมและอัปเดตรายการทันที

หมายเหตุ:
- การลบชุมนุมไม่สามารถย้อนกลับได้
- ชุมนุมที่มีสถานะ Active จะแสดงบนหน้า `/clubs` สาธารณะ ชุมนุมที่ปิด Active จะไม่แสดง
- สถานะ "เปิดรับสมัคร" จะแสดงเฉพาะเมื่อวันปัจจุบันไม่เกินวันสิ้นสุดการรับสมัครที่กำหนด

### 6.12 รายงานการเช็คชื่อเข้าแถว

URL: `/dashboard/flagpole-report`

Flow การสร้างรายงาน:
```
Admin เลือกห้องเรียน + ช่วงวันที่
        ↓
ระบบดึงข้อมูลการเช็คชื่อจาก Database
        ↓
แสดงรายงานแบบ 2 มุมมอง:
  - History: รายการเช็คชื่อรายวันทั้งหมด
  - Summary: สรุปสถิติรายนักเรียน (รายสัปดาห์ / รายเดือน)
        ↓
Admin ส่งออกเป็นไฟล์ Excel (.xlsx) ได้
```

ขั้นตอน:
1. คลิกเมนู "รายงาน" → "รายงานการเช็คชื่อ" ใน Admin Sidebar
2. เลือกห้องเรียนจาก dropdown (หรือเลือก "ทั้งหมด")
3. กำหนดช่วงวันที่ที่ต้องการ (วันเริ่มต้น – วันสิ้นสุด)
4. คลิกปุ่ม "ค้นหา"
5. ดูข้อมูลในแท็บ "ประวัติ" หรือ "สรุป"
6. เลือกช่วงสรุป (รายสัปดาห์ / รายเดือน) ได้จากแท็บสรุป
7. คลิกปุ่ม "ส่งออก Excel" เพื่อดาวน์โหลดไฟล์ .xlsx

ข้อมูลที่แสดงในรายงาน:
- รายชื่อนักเรียน, ห้อง, วันที่, สถานะการเช็คชื่อ
- กราฟแท่งและกราฟวงกลมแสดงสัดส่วนแต่ละสถานะ
- สรุปสถิติ: จำนวนมา / สาย / ลาป่วย / ลากิจ / ขาด รายนักเรียน

### 6.13 รายงานคะแนนความประพฤติ

URL: `/dashboard/behavior-report`

Flow การสร้างรายงาน:
```
Admin เลือกห้องเรียน + ช่วงวันที่ + ช่วงเวลาสรุป
        ↓
ระบบดึงข้อมูลการปรับคะแนนจาก Database
        ↓
แสดงรายงานแบบ 2 มุมมอง:

  - Summary: สรุปคะแนนรวมรายนักเรียน
        ↓
```

| ปัญหา | สาเหตุที่เป็นไปได้ | วิธีแก้ไข |

| เข้าสู่ระบบไม่ได้ | username หรือรหัสผ่านไม่ถูกต้อง | ตรวจสอบ username และรหัสผ่าน หากลืมให้กดลืมรหัสผ่านเพื่อส่งคำขอรีเซ็ต |

# บทที่ 1 บทนำ

## 1.1 สถาปัตยกรรมระบบ

ระบบจัดการเว็บไซต์และสารสนเทศโรงเรียนพัฒนาขึ้นในรูปแบบเว็บแอปพลิเคชัน โดยใช้สถาปัตยกรรมแบบ 3-Tier Architecture แบ่งออกเป็น 3 ชั้น ได้แก่

ชั้น Presentation Layer (ส่วนหน้าของระบบ)

พัฒนาด้วย React.js เวอร์ชัน 18 ซึ่งเป็น JavaScript Library สำหรับสร้างส่วนติดต่อผู้ใช้แบบ Component-based ร่วมกับ Redux Toolkit สำหรับบริหารจัดการ State ของแอปพลิเคชัน และ Tailwind CSS สำหรับออกแบบส่วนแสดงผล ระบบรันอยู่บน Vite Development Server ที่พอร์ต 5173 เครื่องมือและไลบรารีหลักที่ใช้ในชั้นนี้ประกอบด้วย

| ไลบรารี/เครื่องมือ | บทบาทในระบบ |
|---|---|
| React.js 18 | สร้างส่วนติดต่อผู้ใช้แบบ Component |
| Redux Toolkit | จัดการ State และ API Caching (RTK Query) |
| React Router DOM v6 | จัดการการนำทางระหว่างหน้า (Routing) |
| Tailwind CSS | จัดรูปแบบและออกแบบ UI |
| Ant Design | คอมโพเนนต์ UI สำเร็จรูป |
| Vite | Build Tool และ Development Server |
| Axios | ส่ง HTTP Request ไปยัง Backend API |
| SweetAlert2 | แสดง Dialog และการแจ้งเตือน |
| Recharts | แสดงกราฟและ Chart สำหรับรายงาน |
| Editor.js | Rich Text Editor สำหรับเขียนบทความ |

ชั้น Application Layer (ส่วนประมวลผลของระบบ)

พัฒนาด้วย Node.js และ Express.js Framework สำหรับสร้าง RESTful API โดยมี Middleware จัดการการยืนยันตัวตน (Authentication) และการควบคุมสิทธิ์การเข้าถึง (Authorization) ผ่านกลไก JSON Web Token (JWT) ที่เก็บไว้ใน HttpOnly Cookie อายุ 2 ชั่วโมง (Remember Me: 7 วัน) เพื่อความปลอดภัย นอกจากนี้ยังติดตั้ง Security Middleware ครบชุด ได้แก่ Helmet สำหรับตั้งค่า HTTP Security Headers, express-rate-limit สำหรับป้องกัน Brute-force, Winston สำหรับ Structured Logging พร้อม Log Rotation และระบบ Audit Log บันทึกทุกการเปลี่ยนแปลงข้อมูลสำคัญ ระบบรันอยู่ที่พอร์ต 5000 ไลบรารีหลักที่ใช้ประกอบด้วย

| ไลบรารี/เครื่องมือ | บทบาทในระบบ |
|---|---|
| Node.js | Runtime Environment สำหรับรันโค้ด JavaScript ฝั่งเซิร์ฟเวอร์ |
| Express.js | Framework สำหรับสร้าง RESTful API |
| Prisma ORM | Object-Relational Mapper สำหรับติดต่อฐานข้อมูล |
| JSON Web Token (JWT) | ออกและตรวจสอบ Token เพื่อยืนยันตัวตน (อายุ 2h / 7d) |
| bcrypt | เข้ารหัสรหัสผ่านแบบ One-way Hashing |
| Helmet | ตั้งค่า HTTP Security Headers (XSS, Clickjacking, MIME Sniffing) |
| express-rate-limit | จำกัด Request เพื่อป้องกัน Brute-force Attack |
| Winston | Structured Logging พร้อม Log Rotation รายวัน |
| winston-daily-rotate-file | หมุนเวียนไฟล์ Log อัตโนมัติตามวัน/ขนาดไฟล์ |
| cors | ควบคุม Cross-Origin Resource Sharing Policy |
| Multer | จัดการการอัปโหลดไฟล์ (รูปภาพ) |
| cookie-parser | อ่านและจัดการ Cookie ที่แนบมากับ Request |
| csv-parse | แปลงและประมวลผลไฟล์ CSV สำหรับนำเข้าข้อมูลนักเรียน |

ชั้น Data Layer (ส่วนจัดเก็บข้อมูล)

ใช้ระบบจัดการฐานข้อมูลเชิงสัมพันธ์ MySQL โดยมี Prisma ORM ทำหน้าที่เป็นตัวกลางในการติดต่อระหว่าง Application Layer กับฐานข้อมูล ช่วยให้เขียน Query ได้ในรูปแบบ TypeScript/JavaScript ที่ปลอดภัยและอ่านง่าย โดยไม่ต้องเขียน SQL โดยตรง

สถาปัตยกรรมโดยรวมแสดงได้ดังนี้

```
┌───────────────────────────────────────────────────────────────────┐
│               Presentation Layer (Frontend)                       │
│         React.js + Redux Toolkit + Tailwind CSS                   │
│                    (Port 5173)                                    │
└──────────────────────────┬────────────────────────────────────────┘
                           │ REST API (HTTP/HTTPS)
                           │ JWT Token (HttpOnly Cookie, 2h/7d)
┌──────────────────────────▼────────────────────────────────────────┐
│               Application Layer (Backend)                         │
│          Node.js + Express.js + Prisma ORM (Port 5000)            │
│  ┌─────────────────────────────────────────────────────────────┐  │
│  │              Security Middleware Stack                      │  │
│  │  Helmet (HTTP Headers) │ Rate Limit │ CORS │ JWT Verify     │  │
│  └─────────────────────────────────────────────────────────────┘  │
│  ┌─────────────────────────────────────────────────────────────┐  │
│  │           Logging & Audit                                   │  │
│  │  Winston (Structured Log) │ Audit Log → audit_logs table    │  │
│  └─────────────────────────────────────────────────────────────┘  │
└──────────────────────────┬────────────────────────────────────────┘
                           │ Prisma ORM
┌──────────────────────────▼────────────────────────────────────────┐
│                    Data Layer (Database)                          │
│                    MySQL (Tables + audit_logs)                    │
└───────────────────────────────────────────────────────────────────┘
```

## 1.2 เป้าหมายและขอบเขตของระบบ

### 1.2.1 กลุ่มเป้าหมายของโครงงาน

ระบบจัดการเว็บไซต์และสารสนเทศโรงเรียนพัฒนาขึ้นเพื่อรองรับผู้ใช้งาน 3 กลุ่มหลัก ดังนี้

กลุ่มที่ 1: บุคคลทั่วไป ผู้ปกครอง และนักเรียน

เป็นกลุ่มผู้ใช้งานที่ไม่จำเป็นต้องเข้าสู่ระบบ สามารถเข้าถึงข้อมูลสาธารณะของโรงเรียนได้ทันที ได้แก่ ข้อมูลทั่วไปของโรงเรียน รายชื่อครูและบุคลากร ข่าวสารและบทความ ตารางเรียน รวมถึงการตรวจสอบประวัติการเช็คชื่อเข้าแถวและคะแนนความประพฤติของนักเรียนโดยใช้รหัสนักเรียน

กลุ่มที่ 2: ครูและบุคลากร (บทบาท: teacher)

เป็นกลุ่มผู้ใช้งานที่ต้องเข้าสู่ระบบด้วยบัญชีที่ผู้ดูแลระบบสร้างให้ มีสิทธิ์ใช้งานฟังก์ชันสำหรับการปฏิบัติงานประจำวัน ได้แก่ การเช็คชื่อนักเรียนเข้าแถว การบันทึกคะแนนความประพฤติ การบันทึกการเยี่ยมบ้านนักเรียน และการเขียนบทความ/ข่าวสารที่ต้องรอการอนุมัติจากผู้ดูแลระบบก่อนเผยแพร่

กลุ่มที่ 3: ผู้ดูแลระบบ (บทบาท: admin และ super_admin)

เป็นกลุ่มผู้บริหารระบบที่มีสิทธิ์สูงสุด ทำหน้าที่จัดการข้อมูลทั้งหมดในระบบ ได้แก่ การจัดการบัญชีผู้ใช้ การอนุมัติบทความและคำขอรีเซ็ตรหัสผ่าน การจัดการข้อมูลครูและนักเรียน การจัดการปีการศึกษาและห้องเรียน รวมถึงการดูรายงานต่าง ๆ ของระบบ ทั้งนี้ super_admin จะมีสิทธิ์ครอบคลุมทุกอย่างของ admin และมีสิทธิ์จัดการข้อมูลระดับระบบเพิ่มเติม

### 1.2.2 ขอบเขตของระบบ

ระบบจัดการเว็บไซต์และสารสนเทศโรงเรียนครอบคลุมขอบเขตการทำงาน ดังนี้

ขอบเขตสำหรับบุคคลทั่วไป/นักเรียน/ผู้ปกครอง

1. แสดงข้อมูลทั่วไปของโรงเรียน ได้แก่ ประวัติความเป็นมา วิสัยทัศน์และพันธกิจ โครงสร้างองค์กร และข้อมูลการติดต่อ
2. แสดงรายชื่อครูและบุคลากรทั้งหมด จัดกลุ่มตามกลุ่มสาระการเรียนรู้
3. แสดงรายการบทความและข่าวสารที่เผยแพร่แล้ว พร้อมระบบแสดงความคิดเห็น
4. แสดงตารางเรียนและข้อมูลชุมนุมวิชาการ
5. ตรวจสอบประวัติการเช็คชื่อเข้าแถวโดยกรอกรหัสนักเรียน
6. ตรวจสอบคะแนนความประพฤติโดยกรอกรหัสนักเรียน

ขอบเขตสำหรับครู/บุคลากร

1. เช็คชื่อนักเรียนเข้าแถวรายวัน พร้อมระบุสถานะ (มา/ขาด/ลาป่วย/ลากิจ/สาย)
2. บันทึกคะแนนความประพฤตินักเรียน (หักหรือเพิ่มคะแนน พร้อมระบุเหตุผล)
3. บันทึกการเยี่ยมบ้านนักเรียนพร้อมอัปโหลดภาพหลักฐาน (สูงสุด 5 รูป)
4. เขียน แก้ไข และลบบทความข่าวสารของตนเอง
5. เปลี่ยนรหัสผ่านและจัดการโปรไฟล์ส่วนตัว
6. ขอรีเซ็ตรหัสผ่านผ่านระบบ (ต้องรอการอนุมัติจาก Admin)

ขอบเขตสำหรับผู้ดูแลระบบ (Admin/Super Admin)

1. จัดการบัญชีผู้ใช้งาน (กำหนดบทบาท / ลบบัญชี)
2. อนุมัติหรือปฏิเสธบทความของครู และสร้างบทความเองได้
3. อนุมัติหรือปฏิเสธคำขอรีเซ็ตรหัสผ่าน
4. จัดการข้อมูลครูและบุคลากร (เพิ่ม/แก้ไข/ลบ)
5. จัดการข้อมูลนักเรียน (เพิ่มรายคน / นำเข้า CSV / ระงับ / ส่งออก)
6. จัดการปีการศึกษา ภาคเรียน และห้องเรียน
7. จัดการตารางเรียนและชุมนุมวิชาการ
8. แก้ไขข้อมูลประวัติและ Timeline ของโรงเรียน
9. ดูและส่งออกรายงานการเช็คชื่อ คะแนนความประพฤติ และการเยี่ยมบ้าน

ระบบไม่ครอบคลุม: การชำระค่าธรรมเนียม การสื่อสารผ่านระบบแชท การจัดการเกรดและผลการเรียน และการบริหารทรัพยากรบุคคล

# บทที่ 2 การติดตั้งระบบ

## 2.1 การติดตั้ง Web Server

ระบบใช้ Node.js เป็น Web Server สำหรับรันแอปพลิเคชัน Backend และใช้ Vite สำหรับ Frontend การติดตั้งมีขั้นตอนดังต่อไปนี้

ขั้นตอนที่ 1: ติดตั้ง Node.js

1. เข้าไปที่เว็บไซต์ทางการ https://nodejs.org
2. ดาวน์โหลดเวอร์ชัน LTS (Long Term Support) ที่เหมาะกับระบบปฏิบัติการ
3. ติดตั้งโดยทำตามขั้นตอนของ Installer (กด Next ไปจนเสร็จ)
4. ตรวจสอบการติดตั้งโดยเปิด Command Prompt หรือ Terminal แล้วพิมพ์คำสั่ง

```
node -v
npm -v
```

หากแสดงหมายเลขเวอร์ชัน (เช่น v20.x.x) แสดงว่าติดตั้งสำเร็จ

ขั้นตอนที่ 2: ติดตั้ง Git

1. เข้าไปที่เว็บไซต์ https://git-scm.com/downloads
2. ดาวน์โหลดและติดตั้ง Git ตามระบบปฏิบัติการที่ใช้
3. ตรวจสอบการติดตั้งด้วยคำสั่ง

```
git -v
```

ขั้นตอนที่ 3: ดาวน์โหลดซอร์สโค้ดจาก GitHub

1. เปิด Command Prompt หรือ Terminal
2. ไปยัง Folder ที่ต้องการเก็บโปรเจค เช่น `C:\Projects`
3. รันคำสั่งต่อไปนี้เพื่อ Clone โปรเจคจาก GitHub

```
git clone https://github.com/[ชื่อ-GitHub]/[ชื่อ-Repository].git
cd [ชื่อ-Repository]
```

ขั้นตอนที่ 4: ติดตั้ง Dependencies ของ Backend

1. ไปยัง Folder `backend`

```
cd backend
```

2. รันคำสั่งติดตั้ง Dependencies

```
npm install
```

3. ระบบจะดาวน์โหลดและติดตั้ง Library ทั้งหมดที่กำหนดใน `package.json` ได้แก่ Express, Prisma, JWT, bcrypt, Multer และอื่น ๆ โดยอัตโนมัติ

ขั้นตอนที่ 5: ตั้งค่า Environment Variables ของ Backend

1. ใน Folder `backend` สร้างไฟล์ชื่อ `.env`
2. กรอกค่าการตั้งค่าดังนี้

```
PORT=5000
NODE_ENV=development
DATABASE_URL="mysql://root:[รหัสผ่าน]@localhost:3306/eduweb_project"
JWT_SECRET_KEY=[กุญแจลับสำหรับ JWT เช่น ตัวอักษรและตัวเลขแบบสุ่ม 64 หลัก]
JWT_EXPIRES_IN=7d
```

ขั้นตอนที่ 6: ติดตั้ง Dependencies ของ Frontend

1. เปิด Terminal ใหม่ ไปยัง Folder `frontend`

```
cd frontend
```

2. รันคำสั่งติดตั้ง

```
npm install
```

3. สร้างไฟล์ `.env` ใน Folder `frontend` โดยกรอกค่าดังนี้

```
VITE_API_BASE_URL=http://localhost:5000/api
```

## 2.2 การติดตั้งฐานข้อมูล MySQL

ขั้นตอนที่ 1: ดาวน์โหลดและติดตั้ง MySQL

1. เข้าไปที่เว็บไซต์ https://dev.mysql.com/downloads/installer/
2. เลือกดาวน์โหลด MySQL Installer สำหรับ Windows (หรือเวอร์ชันที่เหมาะกับระบบปฏิบัติการ)
3. รันไฟล์ติดตั้ง เลือก Setup Type เป็น "Developer Default" หรือ "Server only"
4. ทำตามขั้นตอน Installer จนครบ รวมถึงการตั้งรหัสผ่านสำหรับ root user (สามารถเว้นว่างได้สำหรับการพัฒนา)
5. ตรวจสอบการติดตั้งโดยเปิด MySQL Command Line Client หรือเปิด MySQL Workbench

ขั้นตอนที่ 2: สร้างฐานข้อมูล

1. เปิด MySQL Command Line Client หรือเชื่อมต่อผ่าน MySQL Workbench
2. รันคำสั่ง SQL เพื่อสร้างฐานข้อมูล

```sql
CREATE DATABASE eduweb_project
  CHARACTER SET utf8mb4
  COLLATE utf8mb4_unicode_ci;
```

3. ตรวจสอบว่าสร้างสำเร็จด้วยคำสั่ง

```sql
SHOW DATABASES;
```

ขั้นตอนที่ 3: สร้างตารางและข้อมูลเริ่มต้นด้วย Prisma

ระบบใช้ Prisma ORM ในการจัดการโครงสร้างฐานข้อมูล ให้รันคำสั่งต่อไปนี้ใน Folder `backend`

1. รัน Migration เพื่อสร้างตารางทั้งหมดตาม Schema ที่กำหนด

```
npx prisma migrate dev --name init
```

หรือหากต้องการ Reset ฐานข้อมูลและเพิ่มข้อมูลตัวอย่าง (Seed Data)

```
npx prisma migrate reset
```

คำสั่งนี้จะ:
- ลบข้อมูลเก่าทั้งหมด
- สร้างตารางใหม่ตาม Schema
- รัน Seed Script เพื่อเพิ่มข้อมูลเริ่มต้น เช่น บัญชีผู้ดูแลระบบ (super_admin)

2. ตรวจสอบโครงสร้างฐานข้อมูลผ่าน Prisma Studio (GUI)

```
npx prisma studio
```

เปิดเบราว์เซอร์ไปที่ `http://localhost:5555` จะเห็นตารางและข้อมูลทั้งหมด

ตารางหลักในฐานข้อมูล

| ชื่อตาราง | ข้อมูลที่จัดเก็บ |
|---|---|
| users | บัญชีผู้ใช้งานทั้งหมดในระบบ |
| teachers | ข้อมูลครูและบุคลากร |
| students | ข้อมูลนักเรียน |
| classschedules | ตารางเรียนรายห้อง |
| flagpoleattendance | ประวัติการเช็คชื่อเข้าแถว |
| studentbehaviorscores | คะแนนความประพฤตินักเรียน |
| homevisits | บันทึกการเยี่ยมบ้านนักเรียน |
| blogs | บทความและข่าวสาร |
| academicclubs | ข้อมูลชุมนุมวิชาการ |
| school_info | ข้อมูลทั่วไปของโรงเรียน |

## 2.3 การติดตั้งเว็บแอปพลิเคชันบน Web Server

หลังจากติดตั้ง Node.js, MySQL และกำหนดค่าตัวแปรสภาพแวดล้อมเรียบร้อยแล้ว ให้ติดตั้งและเปิดใช้งานเว็บแอปพลิเคชันตามขั้นตอนต่อไปนี้

ขั้นตอนที่ 1: Clone โค้ดจาก GitHub

ซอร์สโค้ดของระบบจัดเก็บบน GitHub ให้ดาวน์โหลดโค้ดล่าสุดด้วยคำสั่ง

```
git clone https://github.com/[ชื่อ-GitHub]/[ชื่อ-Repository].git
```

หากมีโค้ดอยู่แล้วและต้องการอัปเดตเป็นเวอร์ชันล่าสุด ให้ใช้คำสั่ง

```
git pull origin main
```

ขั้นตอนที่ 2: เริ่มต้น Backend Server

1. เปิด Terminal ไปยัง Folder `backend`
2. รันคำสั่งสำหรับโหมด Development

```
npm run dev
```

หรือสำหรับโหมด Production

```
npm start
```

3. เมื่อ Server เริ่มต้นสำเร็จ Terminal จะแสดงข้อความ

```
EduWeb-Prototype School Website Server is Running...!
Server running on port 5000
```

4. ตรวจสอบโดยเปิดเบราว์เซอร์ไปที่ `http://localhost:5000` หากแสดงข้อความ "Server is Running" แสดงว่า Backend พร้อมใช้งาน

ขั้นตอนที่ 3: เริ่มต้น Frontend Application

1. เปิด Terminal ใหม่ ไปยัง Folder `frontend`
2. รันคำสั่ง

```
npm run dev
```

3. เมื่อเริ่มต้นสำเร็จ Terminal จะแสดง URL สำหรับเข้าถึงเว็บแอปพลิเคชัน

```
VITE v6.x.x  ready in xxx ms
➜  Local:   http://localhost:5173/
➜  Network: http://[IP ของเครื่อง]:5173/
```

4. เปิดเบราว์เซอร์ไปที่ `http://localhost:5173` ระบบจะแสดงหน้าหลักของเว็บไซต์โรงเรียน

ขั้นตอนที่ 4: ตรวจสอบการทำงานของระบบ

| รายการตรวจสอบ | URL/คำสั่ง | ผลที่คาดหวัง |
|---|---|---|
| Backend API | `http://localhost:5000` | แสดงข้อความ "Server is Running" |
| Frontend | `http://localhost:5173` | แสดงหน้าหลักเว็บไซต์โรงเรียน |
| ฐานข้อมูล | `npx prisma studio` | แสดง GUI ตารางข้อมูลที่ `http://localhost:5555` |
| ทดสอบ API | `http://localhost:5000/api/blogs` | แสดงข้อมูล JSON ของบทความ |

ขั้นตอนที่ 5: เข้าสู่ระบบครั้งแรกด้วยบัญชี Super Admin

หลังจากรัน Seed Data แล้ว ระบบจะมีบัญชี Super Admin สำเร็จรูปสำหรับใช้งานครั้งแรก สามารถเข้าสู่ระบบได้ที่ `http://localhost:5173/login` ด้วยข้อมูลบัญชีที่กำหนดในไฟล์ seed จากนั้นสร้างบัญชีผู้ดูแลระบบและครูเพิ่มเติมตามความต้องการผ่าน Admin Dashboard

# บทที่ 4 บทสรุป

ระบบจัดการเว็บไซต์และสารสนเทศโรงเรียนที่พัฒนาขึ้นนี้เป็นระบบเว็บแอปพลิเคชันแบบครบวงจร (Full-Stack Web Application) ที่ตอบสนองความต้องการของโรงเรียนในยุคดิจิทัล โดยรวบรวมฟังก์ชันการทำงานที่หลากหลายไว้ในระบบเดียว ตั้งแต่การแสดงข้อมูลสาธารณะของโรงเรียน การบริหารจัดการงานประจำวันของครู ไปจนถึงการบริหารระบบในระดับผู้ดูแล

ระบบนี้พัฒนาบนสถาปัตยกรรม 3-Tier โดยใช้เทคโนโลยีที่ทันสมัยและได้รับการยอมรับในวงการพัฒนาซอฟต์แวร์ปัจจุบัน ได้แก่ React.js สำหรับส่วนหน้าของระบบ Node.js พร้อม Express.js สำหรับส่วนประมวลผล และ MySQL ร่วมกับ Prisma ORM สำหรับจัดการฐานข้อมูล การออกแบบระบบยึดหลักความปลอดภัย โดยใช้ JWT Token ที่เก็บใน HttpOnly Cookie เพื่อป้องกัน XSS การเข้ารหัสรหัสผ่านด้วย bcrypt และการตรวจสอบ CORS อย่างเข้มงวด

ระบบสามารถรองรับผู้ใช้งาน 3 กลุ่มได้อย่างมีประสิทธิภาพ กลุ่มบุคคลทั่วไป ผู้ปกครอง และนักเรียน สามารถเข้าถึงข้อมูลสาธารณะของโรงเรียนได้ทันทีโดยไม่ต้องเข้าสู่ระบบ รวมถึงตรวจสอบประวัติการเช็คชื่อและคะแนนความประพฤติด้วยรหัสนักเรียน กลุ่มครูและบุคลากรสามารถปฏิบัติงานประจำวันผ่านระบบได้อย่างสะดวก ทั้งการเช็คชื่อเข้าแถว การบันทึกคะแนนความประพฤติ การบันทึกการเยี่ยมบ้าน และการเขียนบทความข่าวสาร ส่วนกลุ่มผู้ดูแลระบบมีเครื่องมือสำหรับบริหารจัดการข้อมูลทั้งหมดครบถ้วน ตั้งแต่การจัดการบัญชีผู้ใช้ การอนุมัติเนื้อหา การจัดการข้อมูลบุคลากรและนักเรียน ไปจนถึงการออกรายงานในรูปแบบไฟล์ Excel

จุดเด่นของระบบนี้คือการออกแบบที่ยึดผู้ใช้งานเป็นศูนย์กลาง (User-Centric Design) ทำให้แต่ละกลุ่มผู้ใช้เข้าถึงเฉพาะข้อมูลและฟังก์ชันที่จำเป็น ระบบรองรับการแสดงผลบนทุกขนาดหน้าจอ (Responsive Design) ตั้งแต่สมาร์ทโฟนจนถึงคอมพิวเตอร์ตั้งโต๊ะ และมีการแจ้งเตือนและ Flow อนุมัติที่ชัดเจน ส่งผลให้กระบวนการทำงานระหว่างครูและผู้ดูแลระบบเป็นไปอย่างราบรื่น

โดยสรุป ระบบที่พัฒนาขึ้นสามารถตอบโจทย์เป้าหมายหลัก คือการยกระดับการบริหารจัดการข้อมูลโรงเรียนให้มีความเป็นระบบ โปร่งใส และเข้าถึงได้สะดวกสำหรับทุกฝ่ายที่เกี่ยวข้อง ทั้งนักเรียน ผู้ปกครอง ครู และผู้บริหาร อันจะนำไปสู่การพัฒนาคุณภาพการศึกษาอย่างยั่งยืน
